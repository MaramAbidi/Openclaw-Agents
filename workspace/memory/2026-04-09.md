### Send Email Skill Details
The "send-email" skill in OpenClaw is designed to send emails via SMTP using a Python script. Here’s an overview:

**Purpose**: Send emails, including attachments, through an SMTP server with runtime-injected credentials.

**Configuration**: Set up `skills.entries.send-email.env` in `~/.openclaw/openclaw.json`. Required variables:
- `EMAIL_SMTP_SERVER`: SMTP server (e.g., `smtp.163.com` for 163.com, `smtp.gmail.com` for Gmail, etc.)
- `EMAIL_SMTP_PORT`: Port (e.g., `465` for SSL or `587` for TLS)
- `EMAIL_SENDER`: Email address
- `EMAIL_SMTP_PASSWORD`: Authorization code / app-specific password

**Commands**:
1. Sending plain email:
   ```bash
   python3 ~/.openclaw/workspace/skills/send-email/send_email.py "recipient@example.com" "Subject" "Body text"
   ```
2. Sending email with attachments:
   ```bash
   python3 ~/.openclaw/workspace/skills/send-email/send_email.py "recipient@example.com" "Subject" "Body text" "/path/to/file.pdf"
   ```

**Best Practices**:
- Never manually read configuration files (ensures credentials stay secure). The environment is injected automatically.
- Use proper SMTP settings relevant to the email provider (e.g., `smtp.gmail.com` for Gmail).
- Validate the settings if authentication or connection issues arise.

This skill is highly useful for automated notifications and email-based workflows within OpenClaw.
